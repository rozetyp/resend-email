Security & Trust

Your API Key is Safe

Your key is used only in your browser. It's sent directly to Resend's API in the Authorization header and never stored, logged, or persisted anywhere.

Your Email Content is Never Stored

There is no database. Email bodies are forwarded to Resend and discarded immediately after sending. No history. No tracking.

Stateless Proxy Design

Resend Pad is just a thin proxy: it receives your request, adds your API key, forwards to Resend, returns the response. Nothing stored in between.

FAQ

Is this free?

Yes. Resend Pad is completely free. (Resend charges for emails sent, but that's independent of Resend Pad.)

Do I need to create an account?

No. Just paste your Resend API key and start sending. No signup, no email, no accounts.

Can I use this for production emails?

Yes, but you're responsible for domain setup (SPF/DKIM) and compliance with anti-spam policies. Best used for testing and one-off sends.

Can I self-host Resend Pad?

Yes. It's a single Node.js proxy + HTML file. Deploy to Railway, Vercel, or your own server in minutes.

What API key do I need?

You need a "Full Access" or "Sending" key that can list domains. "Sending Only" keys are limited. Get one at resend.com/api-keys.

Common Questions

How do I send email from my domain without code?

Verify your domain in Resend (add SPF/DKIM records), get an API key, paste it into Resend Pad, select your domain, and send. No coding required.

Yes. Your key stays in your browser, sent directly to Resend in the Authorization header only. Never stored, logged, or persisted.
